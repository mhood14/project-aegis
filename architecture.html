Architecture

Zero Trust Azure design with explicit trust separation.

The architecture keeps application, AI, secrets, and data responsibilities separated so security decisions are enforced by the platform and the application rather than casually delegated to the model layer.

Reference architecture diagram

Open full diagram

Network pattern

  • Hub VNet: central routing, NVA traffic forwarding, private DNS, and shared control-plane functions.
  • App spoke: App Service with VNet integration and isolated workload placement.
  • AI spoke: Azure OpenAI and Azure AI Search via private endpoint paths.
  • Data spoke: Storage account access isolated from the public internet.

Control intent

  • Keep model access private and application-mediated.
  • Use bidirectional peering to preserve necessary internal reachability between spokes and hub.
  • Keep identity, data, and AI controls from collapsing into a single boundary.
  • Preserve operational visibility through centralized logging and diagnostics.

Flow

How data and trust move through the system.

01

User identity is established

Entra ID authenticates the user and Conditional Access adds MFA and policy enforcement.

02

App enforces authorization

The backend validates scope and access boundaries before any retrieval or upload action is allowed.

03

Documents are stored and prepared

Uploads are tagged, chunked, and stored with metadata before being made available for retrieval.

04

Scoped retrieval feeds Azure OpenAI

Only the filtered context needed to answer the question is forwarded into the model path.
