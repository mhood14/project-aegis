Project Overview

What this project demonstrates

Cloud Security Architecture

Designed a hub-and-spoke Azure environment with separated app, AI, data, and control-plane responsibilities, using private endpoints, private DNS, and managed identity.

Secure AI Implementation

Built a secure document Q&A platform using Azure OpenAI and Azure AI Search with scope-aware retrieval, prompt hardening, and document handling controls.

Operational Security Mindset

Treated application logs as security telemetry and validated detections in Microsoft Sentinel for authorization failures, content filtering, and suspicious retrieval behavior.

Architecture

Zero Trust Azure design

Network layout

  • Hub: Key Vault, private DNS, shared control-plane services
  • App Spoke: App Service with VNet integration
  • AI Spoke: Azure OpenAI and Azure AI Search private access paths
  • Data Spoke: Storage account private access path

The design separates application, data, and AI responsibilities while keeping identity and network trust decisions out of the model itself.

Core data flow

  1. User authenticates with Entra ID
  2. App enforces authorization and scope boundaries
  3. Documents are uploaded, chunked, tagged, and indexed
  4. Hybrid search retrieves scoped content
  5. Azure OpenAI answers only from retrieved context
  6. Logs flow to Sentinel for monitoring and detections

Security Design

Built like a control system, not just a chatbot

Identity and access

  • Entra ID SSO for user authentication
  • Conditional Access with MFA and blocked legacy auth
  • Server-side scope enforcement before retrieval
  • Security-sensitive scopes restricted to elevated roles

Data and AI safety

  • Documents treated as data, not trusted instructions
  • Chunking reduces blast radius of malicious content
  • Suspicious prompt content can be sanitized before model use
  • Test documents isolated from normal retrieval scopes

Service authentication

  • Managed identity for Storage, Search, OpenAI, and Key Vault access
  • No hardcoded application secrets in code
  • Least-privilege RBAC applied per Azure service

Threats addressed

  • Prompt injection
  • Document poisoning
  • Authorization bypass
  • Data exfiltration through weak retrieval boundaries
  • Model misuse and over-retrieval

Hybrid Search

Keyword precision plus semantic matching

Why hybrid search matters

Keyword search is excellent for exact phrases and known terms. Vector search helps when users ask conceptual or paraphrased questions. Hybrid search combines both, improving retrieval quality without weakening scope filters or trust boundaries.

Implementation summary

  • Azure OpenAI embeddings generated during ingestion
  • Vector field stored in a dedicated Azure AI Search hybrid index
  • Query flow uses text + vector retrieval together
  • Scope and document trust filters remain enforced server-side

Security Telemetry

From logs to detections

Custom detections planned

  • Repeated authorization denial events
  • Suspicious prompt injection or content-filter blocks
  • Unusual fallback retrieval frequency or scope usage

Example KQL ideas

AppServiceConsoleLogs
| where ResultDescription contains '"event_type": "authz_denied"'
AppServiceConsoleLogs
| where ResultDescription contains '"event_type": "content_filter_blocked"'
AppServiceConsoleLogs
| where ResultDescription contains '"event_type": "fallback_used"'
| summarize count() by bin(TimeGenerated, 1d)
