What this project demonstrates

Security themes that matter in real Azure engineering work.

Identity-first design

Authentication and authorization are treated as first-class controls instead of afterthoughts.

Private service architecture

Core services are kept off the public internet wherever possible through private endpoints and DNS.

AI-specific threat awareness

Prompt injection, poisoning, and over-retrieval are treated as engineering risks, not abstract concerns.

Detection mindset

Application and cloud activity are positioned as telemetry sources for Sentinel and operational response.

Resume-ready summary

The short version a recruiter or hiring manager can grasp quickly.

  • Designed and deployed a secure Azure AI platform using hub-and-spoke networking, private endpoints, and managed identity.
  • Implemented Entra ID SSO, Conditional Access, RBAC, and scope-aware access controls for an AI-enabled Flask application.
  • Integrated Azure OpenAI and Azure AI Search into a scoped document retrieval workflow with metadata and chunking controls.
  • Developed an AI-focused threat model covering prompt injection, poisoning, data exfiltration, authorization abuse, and infrastructure risk.
  • Routed security-relevant telemetry into Microsoft Sentinel and Log Analytics to support monitoring and future detections.
  • Packaged the work as a professional case study for Microsoft-aligned cyber security engineering roles.
