Live Demo

The running application and its retrieval flow.

Project Aegis runs as a Flask application that uploads documents, indexes chunked content for hybrid search, retrieves scoped matches, and sends only limited grounded context into Azure OpenAI.

01

Upload and store

Documents are uploaded into Blob Storage as source material for the retrieval pipeline.

02

Chunk and index

Content is prepared into chunks with metadata so hybrid search can combine keyword and vector retrieval.

03

Retrieve with access checks

The app evaluates authenticated user context, applies scope checks, and retrieves only allowed content.

04

Respond with citations

The answer path stays grounded in selected chunks, citations, and logged request activity.

🔎

Hybrid search

The application combines searchable text, vector content, and metadata so retrieval can stay relevant while still supporting filters and document traceability.

🧾

Logging in the app path

Authorization decisions, retrieval behavior, sanitization events, and model-call outcomes are recorded so the workflow can be reviewed later.
