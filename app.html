AI App

The document workflow, search layer, and model call path.

The application side of Project Aegis is a Flask app that accepts documents, processes them into chunks, indexes them for hybrid search, retrieves scoped content, and sends only that limited context to Azure OpenAI.

01

Upload and store

Raw documents are uploaded into Blob Storage, giving the pipeline a durable input source before processing begins.

02

Chunk and prepare

Documents are processed into chunks and prepared so retrieval can stay narrower and more inspectable than one large free-form prompt.

03

Index for hybrid search

The app supports a hybrid index shape that keeps searchable content and vector content together, improving retrieval options without losing filterable metadata.

04

Answer with citations

Only the retrieved chunks selected for the request are sent to the model, and the response is expected to remain grounded in cited chunk IDs.

🧰

Implementation notes reflected from the app

  • Flask provides the web application layer.
  • Managed identity is used through DefaultAzureCredential.
  • Raw and processed document containers separate original files from chunked content.
  • Azure AI Search stores indexed chunks and associated metadata.
  • Azure OpenAI receives the scoped document context, not the entire corpus.
🧭

Guardrails visible in code

  • Retrieved content is treated as untrusted data, not executable instruction.
  • Suspicious prompt-injection style lines can be sanitized before the model call.
  • Authorization decisions are enforced server-side using authenticated user context.
  • Request activity is logged so failures and suspicious events can be investigated later.
